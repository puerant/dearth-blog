**文档版本**：v1.2
**创建日期**：2026-03-08
**项目名称**：dear-blog
**文档类型**：产品需求文档（不含技术实现细节）

---

## 1. 项目概述

### 1.1 产品定位

dear-blog 是一个个人博客系统，内容以**技术文章与生活随笔混合**为主，面向广泛读者群体。博主可通过独立的管理后台创作和管理内容，读者通过门户网站浏览文章。

### 1.2 产品目标

- 为博主提供一套高效、简洁的内容创作与管理工具
- 为读者提供极简风格、阅读体验良好的门户网站
- 支持内容的多维度组织（分类、标签、系列、归档）
- 以静态预渲染方式保证门户网站的加载性能

### 1.3 项目结构

```
dear-blog/
├── frontend/
│   ├── portal/     # 门户网站（面向读者）
│   └── admin/      # 管理后台（面向博主）
├── backend/        # 后端服务
└── docs/           # 公共文档
```

## 2. 用户角色

| 角色 | 描述 | 权限范围 |
|------|------|----------|
| 博主（管理员） | 系统唯一管理员账号 | 完整的内容创作、发布、管理权限；访问 Admin 后台 |
| 访客（读者） | 无需注册的普通读者 | 浏览、阅读门户网站上公开发布的文章 |

> 系统采用**单用户模式**，只有一个博主账号，无需读者注册功能。

## 3. Portal 门户网站

### 3.1 设计风格

- **风格**：极简主义（Minimalism）
- **主色调**：以白色为主，大量留白，细字体
- **色彩切换**：暂不支持暗色/亮色切换，默认亮色
- **参考风格**：类似 Medium、Notion 的简洁阅读体验

### 3.1.1 字体规范（新增，2026-03-10）

- Portal 与 Admin 全站除 Logo 外，统一使用**阿里巴巴普惠体**
- Logo 保持独立品牌字体，不纳入统一字体替换范围
- 正文、标题、导航、按钮、表单、标签等 UI 文本均按统一字体执行
- 字体资源采用**本地托管**方案，不依赖外部 CDN
- 字重使用 `400 / 500 / 700` 三档

### 3.2 移动端适配

- Portal 门户网站需支持移动端响应式自适应布局
- Admin 后台仅需适配桌面端

### 3.3 页面清单

#### 3.3.1 首页

**布局形式**：个人介绍区块 + 文章列表

**首页内容区块（从上到下）**：

1. **个人介绍区**：展示博主头像、名字、一句话简介
2. **精选文章区**：展示博主手动标记为精选的文章（可选模块）
3. **最新文章列表**：按时间倒序展示最新发布的文章，支持分页

**文章列表卡片信息**：
- 文章标题
- 发布日期
- 所属分类
- 标签列表
- 摘要/简介

#### 3.3.2 文章详情页

**URL 规则**：基于文章 slug 唯一标识

**页面内容**：
- 文章标题、发布时间、更新时间
- 所属分类与标签
- 预估阅读时长（根据字数自动计算）
- 文章正文（支持 Markdown 和富文本渲染）
- 侧边栏 / 悬浮目录（TOC，基于文章标题自动生成）
- 上一篇 / 下一篇文章导航（暂不要求）

#### 3.3.3 分类归档页

- 展示所有一级分类（支持多级分类层级）
- 点击分类后，列出该分类下的所有文章
- 展示每个分类下的文章数量

#### 3.3.4 标签归档页

- 以标签云或列表形式展示所有标签
- 点击标签后，列出该标签下的所有文章
- 展示每个标签关联的文章数量

#### 3.3.5 时间归档页

- 按年份 / 月份分组展示所有文章
- 支持时间轴视觉呈现

#### 3.3.6 系列文章页

- 展示博主整理的系列文章集合
- 每个系列包含：系列名称、简介、文章有序列表

#### 3.3.7 关于我页面

- 博主个人介绍（支持富文本/Markdown 内容）
- 可包含：个人简介、技能标签、联系方式等

### 3.4 全局组件

| 组件 | 说明 |
|------|------|
| 导航栏 | 包含站点 Logo/名称、主要页面链接（首页、分类、标签、归档、关于） |
| 页脚 | 版权信息、备案号（可选）、社交媒体链接（可选） |

## 4. Admin 管理后台

### 4.1 认证与安全

- 博主通过账号密码登录 Admin 后台
- 未登录状态访问 Admin 任意页面均跳转至登录页
- 支持登录态持久化（记住登录）
- 支持安全退出登录

### 4.2 功能模块

#### 4.2.1 仪表盘（Dashboard）

- 关键数据概览：总文章数、总访问量、今日访问量
- 最近发布的文章列表（快速入口）
- 访问趋势图表（近 7 天 / 近 30 天）

#### 4.2.2 文章管理

**文章列表**：
- 展示所有文章（含草稿、已发布、已归档状态）
- 支持按状态、分类、标签筛选
- 支持按标题关键词搜索
- 支持批量操作（批量发布、批量删除）

**文章编辑器**：
- 支持 **Markdown 编辑模式**：纯文本 Markdown，实时预览
- 支持**富文本编辑模式**（所见即所得），两种模式可切换
- 文章元信息配置：
  - 标题
  - Slug（URL 唯一标识，支持自动生成）
  - 摘要/简介
  - 封面图片
  - 所属分类（单选）
  - 标签（多选）
  - 所属系列（可选）
  - 系列排序编号（所属系列时有效）
  - 是否精选
  - 发布状态（草稿 / 立即发布 / 定时发布）
  - 自定义发布时间
- 草稿自动保存（防止内容丢失）
- 支持文章预览

**文章状态流转**：

```
草稿 → 已发布 → 已归档
草稿 → 已发布
已发布 → 草稿（取消发布）
已发布 → 已归档
```

#### 4.2.3 分类管理

- 支持创建、编辑、删除分类
- 支持多级分类（至少两级：父分类 → 子分类）
- 分类字段：名称、Slug、描述、父分类、排序权重
- 删除分类前需确认（若分类下有文章，需先迁移或强制删除提示）

#### 4.2.4 标签管理

- 支持创建、编辑、删除标签
- 标签字段：名称、Slug、描述
- 展示每个标签关联的文章数

#### 4.2.5 系列管理

- 支持创建、编辑、删除系列
- 系列字段：名称、Slug、简介、封面图
- 支持调整系列内文章的排列顺序

#### 4.2.6 媒体管理

- 支持上传图片、文件（常见格式：JPG、PNG、GIF、WebP、PDF 等）
- 展示已上传的媒体文件（网格/列表视图）
- 支持复制文件访问链接（用于在文章中引用）
- 支持删除媒体文件
- 在文章编辑器中可直接调用媒体库选择图片

#### 4.2.7 访问统计

- 总访问量（PV / UV）概览
- 按天、周、月的访问趋势折线图
- 文章访问排行榜（Top 10）
- 访客来源概览（可选，按引用来源统计）

#### 4.2.8 个人设置

- 博主账号信息修改（昵称、头像、一句话简介）
- 密码修改
- 博客基础配置（站点名称、站点简介、社交链接等）

## 6. 系统架构概述

### 6.1 整体架构

```
读者浏览器
    │
    ▼
Portal 门户（静态预渲染）
    │
    ▼
Backend API 服务
    │
    ├── 内容接口（文章、分类、标签、系列）
    ├── 媒体接口（文件上传/访问）
    ├── 统计接口（访问量记录）
    └── 管理接口（Admin 专用，需认证）
    │
    ▼
数据库 + 文件存储

博主浏览器
    │
    ▼
Admin 管理后台（动态渲染）
    │
    ▼
Backend API 服务（带认证）
```

### 6.2 渲染策略

| 应用 | 渲染策略 | 说明 |
|------|----------|------|
| Portal 门户 | 静态预渲染（SSG） | 文章内容预先生成静态 HTML，保证加载速度 |
| Admin 后台 | 客户端渲染（CSR） | 动态交互为主，无 SEO 需求 |

### 6.3 部署方式

- **部署目标**：自托管服务器
- Portal 和 Admin 前端作为静态文件/应用部署
- Backend 作为独立服务运行
- 文件存储使用服务器本地存储（或后期扩展至对象存储）

## 7. 非功能性需求

### 7.1 性能

- Portal 首页核心内容加载时间 < 2s（普通网络环境）
- 文章详情页通过静态预渲染保证快速响应

### 7.2 安全性

- Admin 后台所有接口必须经过身份认证
- 密码存储使用加密哈希，不明文存储
- 防止常见 Web 攻击（XSS、CSRF、SQL 注入等）

### 7.3 可用性

- Portal 与 Admin 均需正常运行于主流现代浏览器（Chrome、Firefox、Safari、Edge 最新版）
- Portal 支持移动端（iOS Safari、Android Chrome）响应式布局

### 7.4 可维护性

- 代码按功能模块清晰划分
- 后端提供 RESTful API，前后端分离

## 2. 前端选型

### 2.1 Portal 门户（frontend/portal）

| 分类 | 选型 | 版本要求 | 选择理由 |
|------|------|----------|----------|
| 核心框架 | **VitePress** | 最新稳定版 | Vue 官方 SSG 方案，内置 Vue 3 + Vite，零配置开箱即用，天然支持静态预渲染，适合博客场景 |
| 语言 | **TypeScript** | 5.x | 类型安全，提升代码可维护性，与 Vue 3 的 Composition API 配合完善 |
| CSS 方案 | **Tailwind CSS** | 3.x | 原子化 CSS，灵活高效，适合极简风格定制，无多余样式冗余 |
| Markdown 渲染 | **markdown-it** | 最新稳定版 | 高性能 Markdown 解析器，插件生态丰富，支持代码高亮、数学公式等扩展 |
| 代码高亮 | **Shiki** | 最新稳定版 | VitePress 内置集成，支持 VS Code 主题，构建时渲染无运行时开销 |
| 状态管理 | **Pinia** | 2.x | Portal 状态较简单，Pinia 轻量直观，与 Vue 3 Composition API 原生契合 |
| HTTP 请求 | **Axios** | 1.x | 拦截器、取消请求等功能完善，SSG 构建阶段数据拉取时使用 |
| 路由 | **Vue Router** | 4.x | VitePress 内置，无需单独配置 |
| 渲染策略 | **SSG（静态预渲染）** | — | 构建时生成静态 HTML，无服务端压力，加载速度快，适合博客内容更新频率 |

### 2.2 Admin 管理后台（frontend/admin）

| 分类 | 选型 | 版本要求 | 选择理由 |
|------|------|----------|----------|
| 核心框架 | **Vue 3** | 3.x | 用户指定，Composition API 开发体验优秀 |
| 构建工具 | **Vite** | 5.x | 用户指定，极速热更新，开发体验佳 |
| 语言 | **TypeScript** | 5.x | 类型安全，Admin 表单、接口较多，TS 减少低级错误 |
| UI 组件库 | **Element Plus** | 2.x | Vue 生态最主流组件库，表单、表格、对话框等 Admin 常用组件完备 |
| CSS 方案 | **Element Plus 内置 + SCSS** | — | 与 Element Plus 主题定制配合，局部样式使用 SCSS 嵌套语法 |
| 状态管理 | **Pinia** | 2.x | 轻量、支持 DevTools 调试，Token 及用户信息管理使用 |
| HTTP 请求 | **Axios** | 1.x | 统一请求拦截（自动注入 JWT Token、错误统一处理）|
| 路由 | **Vue Router** | 4.x | 前端路由，支持路由守卫实现登录鉴权 |
| 文章编辑器 | **ByteMD** | 最新稳定版 | 支持 Markdown 编辑+实时预览，插件机制灵活，可扩展代码高亮、数学公式、图表等 |
| 富文本模式 | **ByteMD 富文本插件 / Tiptap** | — | 在 ByteMD 基础上扩展所见即所得能力，或集成 Tiptap 作为富文本备选 |
| 渲染策略 | **CSR（客户端渲染）** | — | Admin 无 SEO 需求，CSR 开发简单，动态交互体验更好 |
| 图表库 | **ECharts** | 5.x | 访问统计图表展示，功能强大，与 Vue 3 有官方集成包（vue-echarts）|

### 2.3 前端公共规范

| 分类 | 工具 | 说明 |
|------|------|------|
| 代码规范 | ESLint + Prettier | 统一代码风格，两个前端项目共用配置 |

## 3. 后端选型

### 3.1 核心框架与语言

| 分类 | 选型 | 版本要求 | 选择理由 |
|------|------|----------|----------|
| 语言 | **Java** | 17 LTS | Spring Boot 3 要求最低 Java 17，LTS 版本长期稳定 |
| 框架 | **Spring Boot 3** | 3.x | 用户指定，自动配置、生态完善、社区活跃 |
| ORM | **MyBatis Plus** | 3.x | 用户指定，在 MyBatis 基础上封装 CRUD，减少重复 SQL，支持代码生成 |
| 构建工具 | **Maven** | 3.x | 项目标配，依赖管理成熟，插件生态丰富 |
| API 风格 | **RESTful API** | — | 前后端分离架构，接口语义清晰 |

### 3.2 安全认证

| 分类 | 选型 | 版本要求 | 说明 |
|------|------|----------|------|
| 安全框架 | **Spring Security** | 6.x（随 Boot 3）| 与 Spring Boot 3 原生集成，提供请求过滤、权限控制能力 |
| 认证方式 | **JWT（JSON Web Token）** | — | 无状态认证，适合前后端分离；博主登录后获取 Token，后续请求携带 Token 鉴权 |
| Token 存储 | 前端 localStorage / Cookie | — | 由前端负责 Token 持久化 |
| Token 黑名单 | **Redis** | — | 登出时将 Token 存入 Redis 黑名单，实现真正意义上的登出失效 |

### 3.3 数据存储

| 分类 | 选型 | 版本要求 | 选择理由 |
|------|------|----------|----------|
| 主数据库 | **MySQL** | 8.x | 关系型数据库首选，社区成熟，与 MyBatis Plus 配合完美 |
| 数据库连接池 | **HikariCP** | Spring Boot 内置 | Spring Boot 默认连接池，高性能轻量 |
| 数据库版本管理 | **Flyway** | — | 数据库 Schema 版本化管理，团队协作和部署时自动执行迁移脚本 |
| 缓存 | **Redis** | 7.x | 用于登录 Token 黑名单、热门文章访问计数缓冲等 |
| 文件存储 | **本地文件系统** | — | 媒体文件存储在服务器磁盘，通过 Nginx 直接提供静态文件访问 |

### 3.4 API 文档

| 分类 | 选型 | 版本要求 | 说明 |
|------|------|----------|------|
| API 文档 | **SpringDoc OpenAPI 3** | 2.x | 基于注解自动生成 OpenAPI 3.0 规范文档，内置 Swagger UI 可视化界面 |

### 3.5 其他工具库

| 分类 | 选型 | 说明 |
|------|------|------|
| 工具类 | **Hutool** | 国内流行 Java 工具库，提供字符串、日期、加密等常用工具 |

## 4. 基础设施选型

### 4.1 Web 服务器

| 选型 | 说明 |
|------|------|
| **Nginx** | 负责：① Portal 静态文件托管；② Admin 静态文件托管；③ 反向代理后端 API 服务；④ HTTPS 终止；⑤ 媒体文件静态访问 |

**Nginx 职责架构示意：**
```
客户端请求
    │
    ▼
  Nginx
    ├── /          → Portal 静态文件目录
    ├── /admin     → Admin 静态文件目录
    ├── /api       → 反向代理 → Spring Boot 服务（8080端口）
    └── /uploads   → 本地媒体文件目录
```

### 4.2 容器化

| 选型 | 说明 |
|------|------|
| **Docker** | 容器化运行后端服务、MySQL、Redis、Nginx |
| **docker-compose** | 编排多容器服务，一键启动/停止整个应用环境 |

**容器编排服务清单：**

| 服务 | 容器 | 说明 |
|------|------|------|
| nginx | nginx:alpine | Web 服务器 + 反向代理 |
| backend | 自定义镜像 | Spring Boot JAR 包 |
| mysql | mysql:8 | 主数据库 |
| redis | redis:7-alpine | 缓存服务 |

## 6. 技术选型决策摘要

| 层次 | 关键决策 | 决策理由 |
|------|----------|----------|
| Portal 渲染 | SSG 静态预渲染 | 博客内容变更频率低，静态化后加载快，服务器压力小 |
| Portal 框架 | VitePress | 内置 Vue3+Vite，对 Markdown 和静态生成支持最好 |
| Admin 框架 | Vue 3 + Vite + Element Plus | 组件库丰富，表单/表格组件成熟，Admin 开发效率高 |
| 编辑器 | ByteMD | Markdown 为主，支持插件扩展，满足技术博客代码高亮等需求 |
| 后端认证 | JWT + Spring Security | 无状态适合前后端分离，结合 Redis Token 黑名单实现可靠登出 |
| 数据库 | MySQL 8 + Redis | 关系型数据存储 + 内存缓存，主流生产级方案 |
| 文件存储 | 本地文件系统 | 个人博客规模，本地存储简单可靠，后期可无缝迁移至对象存储 |
| 部署 | Docker + docker-compose | 环境一致性好，部署简单，一键启动整套服务 |
